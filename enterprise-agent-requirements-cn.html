企业级智能客服Agent系统

需求规格说明书 v1.0

基于LangGraph的生产级对话系统

一、项目概述

构建一个企业级智能客服Agent系统，能够自动处理客户咨询、订单查询、投诉建议等业务，支持人工介入审批，全链路可追踪。

核心目标

  • 7x24小时自动响应客户咨询
  • 多意图识别与智能路由
  • 知识库RAG精准回答
  • 低置信度自动转人工
  • 多轮对话记忆上下文
  • 全链路追踪与监控

技术栈

组件技术选型
编排框架LangGraph
大语言模型GPT-4o / Claude 3.5
向量存储Chroma / FAISS / Pinecone
持久化PostgreSQL / Redis
追踪监控LangSmith
部署Docker

二、功能需求

P0 核心功能（必须实现）

REQ-001: 意图识别与路由

描述：对用户输入进行意图分类，路由到对应的处理分支

意图类型：

  • faq - 常见问题咨询（产品功能、使用方法、退换货政策等）
  • order - 订单查询（订单状态、物流信息、退款进度等）
  • complaint - 投诉建议
  • escalation - 要求转人工

实现要求：

  • 使用LLM进行零样本分类
  • 返回置信度分数（0-1）
  • 置信度低于阈值时自动转人工

REQ-002: FAQ知识库问答

描述：基于知识库的RAG问答系统

功能要求：

  • 支持上传PDF、Word、TXT等格式的文档
  • 文档分块、嵌入、存储到向量数据库
  • 语义检索Top-K相关文档
  • 基于检索上下文生成回答
  • 返回引用来源

REQ-003: 状态持久化与多轮对话

描述：支持多轮对话记忆

实现要求：

  • 使用Checkpointer保存状态
  • 相同session_id共享对话历史
  • 支持中断后恢复
  • 消息历史通过Reducer累积
# 状态定义示例 class CustomerState(TypedDict): messages: Annotated[list[str], operator.add] # 累积 intent: str confidence: float

REQ-004: Human-in-the-Loop人工介入

描述：支持人工审批和转人工

触发条件：

  • 用户明确要求转人工
  • 意图识别置信度 < 0.7
  • RAG检索置信度 < 0.6
  • 检测到投诉/负面情绪
  • 涉及退款、注销等高风险操作

实现要求：

  • 使用 interrupt() 暂停执行
  • 保存当前状态到数据库
  • 人工处理后通过 Command(resume=...) 恢复
  • 记录人工处理日志

P1 重要功能

REQ-005: 订单查询Agent

功能要求：

  • 从用户输入中提取订单号（正则/NER）
  • 查询订单状态、物流信息
  • 支持订单取消、改地址等操作
  • 高风险操作需人工审批

REQ-006: 流式输出

实现要求：

  • 支持 stream_mode="messages" 流式LLM输出
  • 支持 stream_mode="updates" 节点更新输出
  • 异步API支持 (astream)

REQ-007: 对话摘要与记忆压缩

实现要求：

  • 当消息历史超过阈值（如20条）时触发摘要
  • 使用LLM生成对话摘要
  • 保留关键信息，压缩历史

P2 增强功能

REQ-008: 多Agent协作

  • 独立的RAG Agent子图
  • 独立的订单Agent子图
  • 主管Agent统一调度

REQ-009: 情感分析

  • 识别正面/中性/负面情绪
  • 负面情绪自动提升优先级
  • 情绪严重时强制转人工

REQ-010: LangSmith全链路追踪

  • 配置 LANGSMITH_TRACING 环境变量
  • 记录每次调用的输入输出
  • 追踪节点执行时间和成本

六、技术实现提示

1. 状态累积（messages）

from typing import TypedDict, Annotated import operator class State(TypedDict): messages: Annotated[list[str], operator.add] # 每次返回 {"messages": ["new"]} 会自动追加到列表

2. 条件路由

def route(state: State) -> str: if state["intent"] == "faq": return "faq_node" elif state["intent"] == "order": return "order_node" else: return "human_node" graph.add_conditional_edges("classifier", route, { "faq_node": "faq_node", "order_node": "order_node", "human_node": "human_node", })

3. 中断与恢复

from langgraph.types import interrupt, Command def approval_node(state: State): approved = interrupt("等待人工审批...") return {"approved": approved} # 恢复执行 result = graph.invoke(Command(resume=True), config=config)

4. Checkpointer持久化

from langgraph.checkpoint.postgres import PostgresSaver checkpointer = PostgresSaver.from_conn_string( "postgresql://user:pass@host:5432/db" ) app = graph.compile(checkpointer=checkpointer) # 使用thread_id区分不同会话 config = {"configurable": {"thread_id": "user-123-session-1"}} result = app.invoke({"input": "hello"}, config=config)
